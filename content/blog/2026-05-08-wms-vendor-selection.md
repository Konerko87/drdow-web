## WMS 廠商很多，但真正適合中小物流的不多

搜尋「WMS 廠商」時，你會看到很多看起來都很完整的倉儲管理系統。功能表上都有入庫、出庫、盤點、儲位、條碼、報表，甚至每一家都會說自己可以客製。

但真正導入後，差距通常不在功能清單，而在現場能不能用。

台灣中小物流的倉庫條件很現實：空間不一定標準、品項資料不一定乾淨、員工不一定熟電腦、客戶規則常常變、ERP 或 TMS 也可能已經存在。這些條件沒處理好，再漂亮的 WMS 也會變成另一套要人工補資料的系統。

以下 8 個標準，可以幫你在評估 WMS 廠商時少踩坑。

## 1. 廠商是否先問你的倉庫流程，而不是先展示功能？

好的 WMS 廠商不會一開始就打開 Demo 說「我們有 200 個功能」。

他應該先問：

- 現在入庫怎麼驗收？
- SKU 有沒有條碼？
- 儲位有沒有編碼？
- 出庫揀貨誰負責？
- 盤點多久做一次？
- 客戶是否要求批號、效期、溫層、序號？
- ERP 或 TMS 現在怎麼運作？

如果廠商不了解你的流程，只展示通用畫面，導入時很可能發現現場用不上。

## 2. 儲位管理能不能從現場開始，不是只在系統裡畫格子

WMS 的核心是儲位管理，但儲位不是系統自己變出來的。

你要先確認倉庫是否有清楚編碼，例如 A01-2-3 代表 A 區、第 01 通道、第 2 排、第 3 層。如果現場沒有編碼，WMS 廠商要能協助規劃，而不是只說「你們把儲位建進去就好」。

評估時可以問：

- 是否協助設計儲位編碼？
- 是否支援區、通道、排、層、位的層級？
- 是否能標記冷藏、常溫、危險品、暫存區？
- 是否能分析熱區和低頻儲位？

如果你還沒建立儲位邏輯，可以先看 [儲位管理怎麼做才有效](/blog/2026-05-01-storage-location-management)。

## 3. 條碼策略是否包含在導入服務裡？

很多 WMS 報價只寫軟體，不寫條碼策略。

但倉庫如果沒有可掃描的標識，WMS 就只是資料庫。導入前一定要問清楚：

- 商品本身有條碼嗎？
- 沒有條碼的品項要如何貼標？
- 棧板、籠車、周轉箱要不要編號？
- 標籤機、耗材、掃描設備由誰提供？
- 條碼規則是否和 ERP 品號一致？

這些會影響導入成本，也會影響現場是否願意用。便宜的報價如果沒含條碼策略，最後常常會在導入時追加成本。

## 4. 手機端操作是否真的能完成作業？

倉庫現場不可能每個動作都回電腦前操作。

入庫、上架、揀貨、複核、盤點，最好都能在手機或掃描器上完成。評估 WMS 時不要只看主管後台，要要求廠商示範現場人員的流程：

1. 掃到貨條碼
2. 系統建議儲位
3. 上架確認
4. 接到揀貨任務
5. 按儲位順序揀貨
6. 掃碼複核
7. 出庫完成

如果現場操作要點很多層選單，員工很快會回到紙本。

## 5. WMS 能不能和 TMS / ERP 串接？

中小物流常見的錯誤，是 WMS 買了之後才發現它和派車、財務沒有接上。

WMS 管的是倉庫，但出庫完成後通常會進入 TMS 派車；貨品收發、客戶請款、成本結算又會進 ERP。三套系統不串，現場就會變成三份資料。

至少要確認：

- WMS 出庫完成能不能推送到 TMS？
- TMS 配送完成能不能回寫狀態？
- ERP 是否能取得入庫、出庫、庫存異動？
- API 是即時、排程，還是只能匯 CSV？

[Dr.Dow WMS](/products/wms) 的定位不是孤立倉儲軟體，而是和 TMS、ERP 一起構成物流營運主幹。相關觀念可以讀 [WMS + TMS + ERP 怎麼串才不會亂](/blog/2026-04-30-wms-tms-erp-integration)。

## 6. 盤點流程是否支援分區、循環盤點？

很多倉庫一年盤一次，盤點時停業 2 到 3 天。WMS 的價值不是讓大盤點稍微快一點，而是讓你可以改成分區盤點、循環盤點。

評估時要問：

- 是否能按區域產生盤點任務？
- 是否能只盤高價品或高異動品？
- 盤點差異是否需要主管審核？
- 差異調整是否留下完整紀錄？
- 盤點結果是否能回 ERP？

如果 WMS 只是讓你輸入盤點結果，沒有任務派發和差異審核，管理價值會有限。

## 7. 報價是否把導入、教育訓練、資料清整講清楚？

WMS 費用不只月費或授權費。

你還要看：

- 初始導入費
- 顧問流程訪談
- 歷史資料匯入
- 條碼設計和標籤設備
- 教育訓練
- API 串接費
- 後續客製和維護費

如果報價單只寫「WMS 系統一套」，沒有列清楚導入內容，後面很容易一直加價。

可以對照這篇：[WMS 倉儲管理系統價格怎麼算](/blog/2026-05-01-wms-price-cost)。

## 8. 資料能不能完整帶走？

這題很少人問，但非常重要。

導入 WMS 後，系統裡會累積你的庫存歷史、出入庫紀錄、盤點差異、儲位異動、客戶規則。如果未來你想換系統，這些資料能不能完整匯出？

合約裡要寫清楚：

- SKU、儲位、庫存、異動紀錄是否可匯出
- 匯出格式是 CSV、Excel 還是 API
- 是否需要額外費用
- 停用後資料保留多久

不要讓自己的倉庫資料被廠商綁住。

## 結論：WMS 廠商要看導入能力，不只看功能數量

適合台灣中小物流的 WMS，要能處理現場條件：儲位還沒整理好、條碼不完整、員工不想用複雜系統、ERP/TMS 已經存在、客戶規則常常變。

所以選 WMS 廠商時，不要只問「有沒有這個功能」，要問「你會怎麼幫我們導入到現場」。
