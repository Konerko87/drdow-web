## 「市面上 WMS 那麼多，到底要怎麼選？」

這是中小物流老闆在評估 WMS 時最常卡住的問題。

報價單從新台幣 50 萬到 800 萬都有人開，功能清單看起來都差不多，但實際導入完才發現：有的根本接不上你現在的 ERP、有的現場員工根本不會用、有的明明買了還是要靠 Excel 補。

選 WMS 不是看廠商投影片，是先把自己問清楚。

## 問題 1：你最痛的是「找不到貨」還是「對不起帳」？

聽起來很像，差很多。

- **找不到貨**：實體在倉裡，但人不知道在哪個儲位 → 你需要的是**儲位管理 + 條碼揀貨**
- **對不起帳**：帳上有 1,200 件，現場數出來只有 1,150 件 → 你需要的是**即時異動同步 + 定期盤點**

很多中小物流以為自己缺前者，盤點才發現是後者。先做一次完整盤點，差異率超過 3% 的話，重點放對帳。

## 問題 2：未來 3 年會不會接 ERP / TMS？

**這題決定 80% 的廠商選擇**。

買得起的單機 WMS 通常無法跟你的 ERP 即時雙向同步——它只能用每天匯出 CSV 上傳的方式做半自動串接。中型物流通常 3 個月內就會發現「為什麼出貨還要先在 WMS 點一次、再到 ERP 點一次」，然後又開始 Excel 接力。

選 WMS 第一天就要問廠商：「你跟我的 ERP 怎麼串？是 API 即時、API 排程、還是 CSV 匯入？」如果答案只有最後一個，這套 WMS 三年後一定會被淘汰。

## 問題 3：現場員工平均年齡幾歲？學歷？

中小物流的倉管常常是 50+ 歲、習慣紙本作業的資深員工。給他們一套需要 3 天訓練、操作 8 個按鈕才能完成揀貨的 WMS，下場通常是「主管報表很漂亮、現場還是用紙本」。

評估 WMS 時務必要求廠商：

- 帶一套能在手機上完整跑完入庫、揀貨、盤點流程的 Demo
- 模擬現場員工拿到手機 5 分鐘內能否完成第一筆操作

如果 Demo 是在筆電上點 PowerPoint 解釋「這個按鈕代表入庫」，這家廠商不會懂中小物流現場。

## 問題 4：倉庫的網路訊號好嗎？

冷凍倉、高架倉、地下室倉常常 wifi 訊號斷斷續續。如果 WMS 只能線上用，掃 10 筆有 3 筆連不上，員工三天就放棄。

問廠商：**是否支援離線作業？暫存的資料如何同步？**

理想答案：手機暫存掃碼資料，回到訊號區自動同步、自動避免重複過帳。

## 問題 5：條碼怎麼貼？

不要小看這題。

- 商品本身有條碼？→ 直接套用即可
- 沒有條碼？→ 要不要設計流通條碼？棧板要不要編號？循環容器（籠車、塑膠籃）要不要追蹤？

這部分通常**不在 WMS 報價單裡**，但會佔導入成本 10-30%。要求廠商提供完整的「條碼策略建議書」，含貼標機、耗材、回收標籤的標準。

## 問題 6：盤點頻率你能不能改？

老一輩物流業習慣「一年盤一次、停業三天」。

WMS 真正的價值不是讓這三天變得更快，是**把盤點從一年一次的大事件，變成每週、每天都可以做的常態**。差異發生當下就抓出來。

但如果你的營運模式（例如代客倉儲、有客戶在現場）讓你**完全不能改盤點頻率**，那 WMS 帶來的 ROI 就會打折。買之前先想清楚。

## 問題 7：合約怎麼結束？

這題沒人問，但最重要。

- 如果三年後你想換系統，**你的庫存歷史資料能不能匯出？格式是什麼？**
- 廠商有沒有「資料攜出條款」？
- 雲端版 vs 自架版差別在哪？

很多老 WMS 廠商不寫這條，你的歷史資料就被綁死。

新一代雲端 WMS（包含 Dr.Dow WMS）會明確寫：**所有客戶資料隨時可以 CSV / JSON 完整匯出**。

## 中小物流的選擇條件清單

把上面 7 題答完後，回頭看自己的需求應該長這樣：

✅ 與 ERP / TMS 即時 API 雙向串接
✅ 手機端可單獨完整作業（不依賴電腦）
✅ 支援離線作業
✅ 雲端 SaaS，月費制，不綁長期合約
✅ 中文介面、台灣物流場景設計
✅ 條碼策略含在導入服務裡
✅ 資料隨時可完整匯出

符合這些條件的選項中，再看價格、看廠商支援能力、看其他客戶實際導入案例。

## 不要被「功能完整」騙了

WMS 廠商最會做的事是把功能列表寫到 200 條，然後畫上勾勾。

但中小物流真正每天會用的功能，**通常不超過 20 條**。多出來的 180 條只是讓系統變慢、員工搞不清楚。

選 WMS 的判斷標準應該是：

- 我每天最痛的 5 件事，這套有沒有解？
- 解的方式是不是現場員工真的能執行？
- 三年後我想升級或換系統，會不會被綁死？
