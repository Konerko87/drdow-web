## 為什麼「明明有庫存表，業務還是不敢報交期」

很多物流公司的倉庫，現場長這樣：

- 主管看的庫存表是昨天夜批跑出來的
- 揀貨員拿著紙本清單在儲位之間繞來繞去
- 盤點要停業三天，Excel 對到天亮
- 業務想接單，先打電話到倉庫問「這個還有沒有」，倉管喊著翻紙箱回答

這不是員工不認真，是**沒有一套系統把現場每一次異動立刻反映到帳上**。

WMS（Warehouse Management System，倉儲管理系統）要解決的，正是這件事：**讓倉庫現場和庫存帳，永遠同步。**

![倉儲庫存儀表板](/screenshots/wms-inventory.png)

## WMS 到底在做什麼？三件核心事

WMS 不是「進階版的 Excel 庫存表」。它是專門管理**倉庫現場作業**的系統，核心做三件事：

### 1. 把每一次「現場動作」變成系統事件

到貨進倉、揀貨出庫、盤點點數、棧板歸還，每一個動作都用條碼或 QR 掃進系統。沒有「我等等再 key 進去」的空間，也就不會有「東西不見了但帳上還在」這種事。

### 2. 把儲位變成可被計算的資料

倉庫不是平面，是立體的：A 區 / B 區 / C 區，每區有排、有層、有位。WMS 用 **區/排/層/位** 四層結構管理儲位，這樣就能：

- 自動指派新到貨該放哪
- 揀貨清單按儲位排序，揀貨員不會走回頭路
- 儲位視覺化地圖一眼看出哪區滿了、哪區還空

![儲位視覺化地圖](/screenshots/wms-locations.png)

### 3. 把庫存從「報表」變成「即時數字」

每次掃碼異動，可用量、安全庫存、在途數立刻更新。業務看到的不是昨天的數字，是現在這一秒的數字。

## WMS、ERP、TMS 到底差在哪？

很多人會問：「我已經有 ERP 了，為什麼還需要 WMS？」

簡單講：

| 系統 | 管什麼 | 主要使用者 |
| --- | --- | --- |
| **ERP** | 帳上有什麼、誰欠誰錢、多少成本 | 老闆、會計、財務 |
| **WMS** | 現場物理上在哪、怎麼動的 | 倉管、揀貨員、盤點員 |
| **TMS** | 貨怎麼出、誰開車、走哪條路 | 調度員、司機 |

ERP 知道「公司有 1,200 件 A 商品」，但**不知道**這 1,200 件分布在哪三個倉、哪幾個儲位、哪些是即將過期、哪些被預留給未出貨訂單。

WMS 補的就是這層「物理現實」。三套系統打通後：

- 倉庫掃碼出貨 → WMS 扣帳 → 自動推 TMS 派車 → 派車完成推 ERP 帳務

從訂單到撥款，每一步都在同一筆事實來源上。

## 把盤點從 3 天變 3 小時，到底是怎麼辦到的

傳統盤點：

1. 公告停業 → 全員到倉庫
2. 紙本清單分區 → 人工點數
3. 點完回辦公室 → key 進 Excel
4. Excel 跟系統對 → 找差異、人工調整
5. 整套跑完三天

WMS 盤點：

1. 系統派任務到手機（日盤、週盤、年盤）
2. 倉管現場用手機掃碼或輸入數量
3. 系統即時比對「系統數 vs 實盤數」
4. 差異報表自動產出，差異品項可即時調整

![手機盤點任務](/screenshots/wms-stocktake.png)

關鍵不是「速度更快」，是**把盤點從一年一次的大事件，變成每週、每天都可以做的常態**。差異發生當下就被抓出，不會累積到年度盤點才爆掉。

## 一個被忽略的價值：棧板與循環容器追蹤

物流業有個很實際的問題：**棧板會跑**。

司機把貨送到客戶，棧板留在客戶那；客戶下次叫貨，棧板可能流到另一個倉。一年下來，幾百張棧板分散在哪裡，沒人說得清。

WMS 把每張棧板用序號管理，從哪裡出、誰帶走、什麼時候歸還，全程留紀錄：

![棧板與循環容器追蹤](/screenshots/wms-pallet.png)

每月盤一次，「在用 1,240、待歸還 87、逾期未還 12、本月遺失 3」一目了然。逾期未還的可以立刻打電話給司機或客戶追，不會等到年底才發現少了一卡車的棧板。

## 哪些公司應該認真評估 WMS

並不是每家公司都馬上需要 WMS。但如果你符合以下兩項以上，就值得認真看：

1. **倉庫超過 200 坪 / 三層以上儲位**：人工記憶容量已經不夠
2. **單日進出貨 100 筆以上**：紙本和 Excel 開始追不上節奏
3. **倉庫不只一個**：多倉之間調貨、轉運靠 LINE 群組
4. **業務常常無法立即報交期**：因為庫存表是昨天的
5. **盤點停業超過一天**：對營運影響太大
6. **棧板、籠車、紙箱循環容器追不到**：每年因為遺失打掉的成本可觀

## 不是所有 WMS 都適合台灣中小物流

市面上 WMS 大多是兩種：

- **國際大廠版本**：功能完整，但動輒七位數導入費，培訓週期半年起
- **單機買斷版本**：便宜，但跟 ERP / TMS 串接困難，資料各管各的

Dr.Dow 的 WMS 走第三條路：**雲端、模組化、可與 TMS / ERP 即時串接、繁中介面、台灣物流場景設計**。

不是要倉管學會「整個 WMS」，而是讓現場作業員打開手機就會用：

- 接到揀貨任務 → 按儲位順序揀貨 → 掃碼複核 → 完成
- 收到盤點任務 → 掃碼或輸入 → 系統自動算差異
- 主管看儀表板 → 在庫量、低庫存警示、進出貨趨勢一頁掌握

把 WMS 從「資訊部專案」變成「現場工具」，倉庫才會真的用起來。
